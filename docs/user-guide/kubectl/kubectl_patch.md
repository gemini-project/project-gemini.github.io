## gemctl patch

Update field(s) of a resource using strategic merge patch.

### Synopsis


Update field(s) of a resource using strategic merge patch

JSON and YAML formats are accepted.

Please refer to the models in https://htmlpreview.github.io/?https://github.com/gemini-project/gemini/blob/release-1.2/docs/api-reference/v1/definitions.html to find if a field is mutable.

```
gemctl patch (-f FILENAME | TYPE NAME) -p PATCH
```

### Examples

```

# Partially update a node using strategic merge patch
gemctl patch node gem-node-1 -p '{"spec":{"unschedulable":true}}'

# Partially update a node identified by the type and name specified in "node.json" using strategic merge patch
gemctl patch -f node.json -p '{"spec":{"unschedulable":true}}'

# Update a container's image; spec.containers[*].name is required because it's a merge key
gemctl patch pod valid-pod -p '{"spec":{"containers":[{"name":"gemini-serve-hostname","image":"new image"}]}}'

# Update a container's image using a json patch with positional arrays
gemctl patch pod valid-pod -type='json' -p='[{"op": "replace", "path": "/spec/containers/0/image", "value":"new image"}]'
```

### Options

```
  -f, --filename=[]: Filename, directory, or URL to a file identifying the resource to update
  -o, --output="": Output mode. Use "-o name" for shorter output (resource/name).
  -p, --patch="": The patch to be applied to the resource JSON file.
      --record[=false]: Record current gemctl command in the resource annotation.
      --type="strategic": The type of patch being provided; one of [json merge strategic]
```
